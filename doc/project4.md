## 1. 서론      
### 1.1 문서의 목적 및 범위
본 문서는 Mini Drive System의 요구사항을 정의하기 위한 문서이다.  
사용자의 요구사항을 기반으로 시스템이 수행해야 할 기능과 품질 속성을 명확히 정의하며, 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항을 포함한다.

---
### 1.2 프로젝트 개요   
1.2.1 프로젝트 정의  
Mini Drive System은 조직 내 파일을 중앙에서 관리하고 사용자 간 파일 공유를 지원하는 클라우드 기반 시스템이다.  

1.2.2 주요 기능 설명   
|ID|	설명  |
|:--|:--|
|F01|	파일 업로드 및 다운로드 기능     |
|F02|	사용자 로그인 및 인증 기능    |
|F03	|파일 공유 기능   |
|F04	|폴더 생성 및 파일 관리 기능  |  
|F05	|파일 검색 기능   |
|F06	|파일 버전 관리 기능|  

---

|:--|:--|

예외 처리 및 권한 상세 기능   

|분류 | 요구사항 | 우선순위 |
| :-- | :-- | :-- | 
| FR-015 | 회원가입 시 이미 등록된 이메일 주소인 경우 시스템은 중복 알림 메시지를 반환한다. | 상 |
| FR-016 | 로그인 실패 시 시스템은 세션을 발급하지 않고 실패 원인(아이디/비밀번호 불일치 등)을 안내한다. | 상 |
| FR-017 | 파일 업로드 중 허용 용량 초과 또는 네트워크 연결 단절 시 업로드를 취소하고 에러 메시지를 표시한다. | 상 |  
| FR-018 | 삭제되었거나 존재하지 않는 파일 링크로 접근 시 시스템은 '파일을 찾을 수 없음(404)' 안내 페이지를 반환한다. | 상 |  
| FR-019 | 시스템이 제공하는 파일 및 폴더의 접근 권한은 '보기 가능(조회/다운로드)'과 '편집 가능(수정/삭제/추가)'으로 세분화하여 설정한다. | 상 |  
| FR-020 | 권한이 없는 사용자가 공유 파일이나 폴더에 접근을 시도할 경우 시스템은 접근 차단 메시지를 반환한다. | 상 |
| FR-021 | 공유 링크의 유효 기간이 만료된 경우 시스템은 접근을 차단하고 만료 안내 페이지를 반환한다. | 중 |
