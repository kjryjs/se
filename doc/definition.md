# 📄 프로젝트 정의서

## (1) 프로젝트 명칭
클라우드 파일 공유 시스템

---

## (2) 개발 시스템 명칭
Mini Drive System

---

## (3) 개발 시스템 설명
Mini Drive System은 조직 내 파일을 중앙에서 관리하고 사용자 간 파일 공유를 지원하는 클라우드 기반 시스템이다.  
사용자는 웹 브라우저를 통해 파일을 업로드, 다운로드 및 관리할 수 있으며, 폴더 구조를 활용해 파일을 체계적으로 정리할 수 있다.  
또한 사용자 간 파일 공유 기능을 통해 협업 효율을 향상시키는 것을 목표로 한다.

---

## (4) 핵심 기능

- 파일 업로드 및 다운로드 기능  
- 폴더 생성 및 파일 정리 기능  
- 사용자 로그인 및 인증 기능  
- 사용자 간 파일 공유 기능  
- 파일 검색 기능  
- 파일 버전 관리 기능  

---

## (5) 기존 유사 소프트웨어 조사 및 분석

### 1. Google Drive
**장점**
- 강력한 검색 기능 제공  
- 실시간 협업 기능 지원  
- 다양한 파일 형식 지원  

**단점**
- 조직 맞춤 기능 구현이 어려움  
- 외부 서비스 의존성 존재  

---

### 2. Dropbox
**장점**
- 간단하고 직관적인 인터페이스  
- 빠른 파일 동기화 속도  

**단점**
- 무료 저장 공간 제한  
- 협업 기능이 상대적으로 부족  

---

### 3. Microsoft OneDrive
**장점**
- Microsoft Office와 강력한 연동  
- 기업 환경에 적합한 보안 기능  

**단점**
- 특정 플랫폼에 의존적  
- 일부 기능 사용 시 성능 저하 가능  

---

### ✔ 종합 분석
기존 클라우드 서비스는 편리한 기능을 제공하지만 조직의 요구사항에 맞춘 커스터마이징이 어렵고, 보안 정책을 완전히 통제하기 어렵다는 한계가 있다.  
따라서 본 프로젝트에서는 조직 환경에 최적화된 파일 관리 시스템을 개발하는 것을 목표로 한다.
