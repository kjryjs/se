# 📄 프로젝트 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트 명
클라우드 파일 공유 시스템 (Mini Drive)

### 1.2 목적
조직 내 파일을 중앙에서 관리하고, 사용자 간 파일 공유를 쉽게 하기 위한 시스템을 개발한다.

---

## 2. 문제 상황

현재 파일 공유 방식의 문제:
- 파일 버전 관리 어려움
- 파일 찾기 어려움
- 외부 공유 시 보안 문제

---

## 3. 유사 소프트웨어 조사

### 3.1 Google Drive
**장점**
- 강력한 검색 기능
- 실시간 협업 가능

**단점**
- 조직 맞춤 기능 제한

---

### 3.2 Dropbox
**장점**
- 사용이 간단함
- 파일 동기화 빠름

**단점**
- 무료 용량 제한

---

### 3.3 OneDrive
**장점**
- MS 서비스와 연동

**단점**
- 특정 환경에 종속적

---

### ✔ 결론
기존 서비스는 편리하지만  
조직에 맞게 기능을 수정하기 어려움 → 자체 시스템 필요

---
