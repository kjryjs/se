# 1. 서론

## 1.1 문서의 목적 및 범위
본 프로젝트 계획서는 Mini Drive System 프로젝트의 개발을 체계적으로 수행하기 위한 관리 기준을 정의한다.  
프로젝트의 일정, 조직, 품질, 산출물, 리스크 등을 명확히 하여 프로젝트 수행의 기준 지표로 활용한다.  

###  범위
- 프로젝트 개요 및 목표
- 개발 계획 및 일정
- 팀 구성 및 역할
- 품질 관리 
- 산출물
- 리스크 관리

---

## 1.2 프로젝트 개요

Mini Drive System은 조직 내부에서 파일을 중앙 관리하고 사용자 간 공유를 지원하는 클라우드 기반 시스템이다.  

웹 기반 환경에서 파일 업로드, 다운로드, 검색, 공유 기능을 제공하여 협업 효율 향상을 목표로 한다.

---

## 1.3 프로젝트 목표
- 조직 맞춤형 파일 관리 시스템 구축  
- 데이터 손실 없는 안정적인 파일 저장  
- 빠른 파일 처리 및 검색 성능 제공  
- 안전한 파일 공유 및 접근 제어 구현  
- 유지보수 및 확장이 용이한 구조 설계  

---

## 1.4 용어 정의
| 용어 | 설명 |
|------|------|
| 클라우드 파일 시스템 | 네트워크 기반 파일 저장 및 공유 시스템 |
| API | 시스템 간 통신 인터페이스 |
| 버전 관리 | 파일 변경 이력 관리 기능 |

---

## 1.5 참조 문서
- 프로젝트 정의서  
- 품질 요소 추정 문서

# 2. 개발 계획

## 2.1 개발 절차 모형
본 프로젝트는 요구사항 변경 가능성이 높고 사용자 피드백 반영이 중요하므로  
Agile 기반 Scrum 모델을 적용한다.  
객체지향 설계를 기반으로 시스템을 개발한다.

---

## 2.2 WBS (Work Breakdown Structure)
프로젝트  
├── 계획  
├── 요구사항 정의  
├── 요구사항 분석  
├── 설계  
├── 개발  
│ ├── 파일 관리 기능  
│ ├── 사용자 인증  
│ ├── 파일 공유  
│ ├── 검색 기능  
│ └── 버전 관리   
├── 테스트  
└── 배포  

---

## 2.3 개발 활동
- 프로젝트 요약
- 프로젝트 계획
- 요구사항 정의 및 분석  
- Product Backlog   
- Sprint Planning  
- 기능 개발 및 테스트  
- Sprint Review  

---

## 2.4 개발 일정

| 단계 | 기간 |
|------|------|
| 프로젝트 계획 | 1주 |
| 요구사항 분석 | 1~2주 |
| 설계 | 3주 |
| 개발 | 4~8주 |
| 테스트 | 9주 |
| 배포 | 10주 |
